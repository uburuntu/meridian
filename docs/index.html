Setup

One command

You need a VPS (Debian/Ubuntu, root SSH access) and a terminal. That's it.

curl -sS https://raw.githubusercontent.com/rmbk/meridian/main/setup.sh | bash

The script asks for your server IP, installs dependencies, configures the proxy, and outputs a QR code. Or pass the IP directly:

curl -sS https://raw.githubusercontent.com/rmbk/meridian/main/setup.sh | bash -s -- YOUR_IP
or with a domain (adds a decoy website)
curl -sS https://raw.githubusercontent.com/rmbk/meridian/main/setup.sh | bash -s -- YOUR_IP --domain example.com

What happens

Under the hood

1
Installs Ansible & qrencode if not already present
2
Downloads the playbook from GitHub (no git needed)
3
Hardens the server — firewall, SSH keys only, BBR congestion control
4
Deploys VLESS+Reality via Docker — looks like microsoft.com to any probe
5
Outputs a QR code and saves an HTML file you can send to anyone
⏱ Safe to re-run at any time. The script is fully idempotent — it picks up where it left off.

Connect

Scan and go

After the script finishes, you get an HTML file with a QR code. Send it to whoever needs it. They scan it, connect, done.

Advanced

More options

Add a decoy website (domain mode)

If you have a domain, the server also serves a fake business website and supports CDN fallback through Cloudflare. Pass --domain to the setup command.

The server uses HAProxy to route traffic by SNI — censors see a legitimate website, your traffic goes through Reality.

Relay chain (IP whitelist bypass)

If direct foreign connections are blocked (IP whitelisting), use a two-server chain: a domestic relay on a whitelisted IP forwards traffic to the exit server abroad.

git clone https://github.com/rmbk/meridian.git && cd meridian cp inventory-chain.yml.example inventory-chain.yml # Edit inventory-chain.yml with both server IPs ansible-playbook -i inventory-chain.yml playbook-chain.yml

The relay accepts plain VLESS+TCP (no TLS needed for domestic traffic) and forwards everything abroad via VLESS+Reality+XHTTP. Local sites are routed directly.

Manual Ansible setup (full control)

If you prefer to manage inventory and variables yourself:

git clone https://github.com/rmbk/meridian.git && cd meridian pip3 install ansible ansible-galaxy collection install -r requirements.yml # Edit inventory.yml with your server IP ansible-playbook playbook.yml

See group_vars/all.yml for all configurable variables (SNI target, ports, client limits, etc.).

How to update

SSH into the server and run:

cd /opt/3x-ui && docker compose pull && docker compose up -d

Or re-run the setup command — it's safe to run repeatedly.
